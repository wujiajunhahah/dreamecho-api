# 🚀 DreamEcho 快速部署到手机

## 最简单的方法（推荐新手）

### 1. 打开 Xcode
```bash
open -a Xcode
```

### 2. 创建新项目
1. 点击 **File > New > Project...**
2. 选择 **iOS > App**
3. 点击 **Next**

### 3. 填写项目信息
- **Product Name**: `DreamEcho`
- **Team**: 选择你的 Apple ID（如果没有，点击 "Add Account..." 添加）
- **Organization Identifier**: `com.你的名字` （例如：`com.wujiajun`）
- **Bundle Identifier**: 会自动生成（例如：`com.wujiajun.DreamEcho`）
- **Interface**: **SwiftUI** ✅
- **Language**: **Swift** ✅
- **Storage**: None
- **Include Tests**: ✅（可选）

### 4. 选择保存位置
保存到：`/Users/wujiajun/dream_to_model_web/ios/`

### 5. 导入源代码
1. 在 Xcode 左侧项目导航器中，**删除** 自动生成的 `ContentView.swift`
2. 打开 Finder，前往 `/Users/wujiajun/dream_to_model_web/ios/DreamEchoApp/Sources/`
3. 选中 `Sources` 文件夹下的**所有文件和文件夹**
4. 拖拽到 Xcode 项目的 `DreamEcho` 文件夹中
5. 在弹出的对话框中：
   - ✅ **Copy items if needed**
   - ✅ **Create groups**
   - ✅ **Add to targets: DreamEcho**
   - 点击 **Finish**

### 6. 配置最小版本
1. 点击 Xcode 左侧的项目名（蓝色图标）
2. 选择 **TARGETS > DreamEcho**
3. 在 **General** 标签页中：
   - 找到 **Minimum Deployments**
   - 改为 **iOS 16.0**

### 7. 连接你的 iPhone/iPad
1. 用数据线连接设备到 Mac
2. 在手机上：
   - 打开 **设置**
   - 滚动到底部 **开发者**（如果没有，继续下一步）
   - 如果弹出提示框"要信任此电脑吗？"，点击 **信任**

3. 如果没有开发者选项：
   - 打开 **设置 > 隐私与安全 > 开发者模式**
   - 打开开发者模式
   - 重启设备

### 8. 运行应用！
1. 在 Xcode 顶部工具栏，点击设备选择器（显示"My Mac (Designed for iPad)"的地方）
2. 选择你连接的 **iPhone** 或 **iPad**
3. 点击 ▶️ **Run** 按钮（或按键盘 **Cmd + R**）
4. 等待编译（首次需要 1-3 分钟）

### 9. 信任开发者（首次运行）
如果手机上显示"无法验证开发者"：
1. 在 iPhone/iPad 上打开 **设置**
2. 进入 **通用 > VPN与设备管理**
3. 找到你的 Apple ID
4. 点击 **信任 "你的Apple ID"**
5. 返回主屏幕，重新打开 DreamEcho

## 常见问题

### ❓ "No account for team"（没有团队账号）
**解决：**
1. 打开 Xcode > **Settings...** (或按 Cmd+,)
2. 选择 **Accounts** 标签
3. 点击左下角 **+** 号
4. 选择 **Apple ID**
5. 登录你的 Apple ID（免费账号即可）

### ❓ 找不到设备
**解决：**
1. 拔掉数据线重新插
2. 在设备上点击"信任此电脑"
3. 在 Xcode 中选择 **Window > Devices and Simulators**
4. 查看设备是否出现

### ❓ 编译错误 "Cannot find type..."
**解决：**
1. 确保所有文件都已正确导入
2. 选择菜单 **Product > Clean Build Folder** (Shift+Cmd+K)
3. 重新运行 (Cmd+R)

### ❓ 应用闪退
**解决：**
1. 查看 Xcode 底部的 **Console** 输出
2. 查找错误信息（通常是红色文字）
3. 最常见原因：后端 API 地址未配置
